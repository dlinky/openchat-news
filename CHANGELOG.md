# Changelog

## v1.1 — 2026-03-06

### 버그 수정
- **업로드 완료 후 모달 자동 닫힘 제거** — 완료 화면은 사용자가 직접 닫아야 닫힘

### 로직 개선
- **채팅방 이름 끝 숫자 제거** (`parser.py`) — 참여인원 변경으로 인한 중복 채팅방 생성 방지
  예: `"스터디방 42"` → `"스터디방"`
  > 기존 DB 정리: `backend/migrate_v1_1.sql` 실행 필요
- **주/월간 생성 배치 방식** (`generate.py`) — 주간/월간 생성 시 하위 일간/주간 다이제스트를 자동으로 생성
- **주/월간 생성 거부 조건** — `global_max_date < 기간 마지막 날`이면 생성 거부, 오류 메시지 표시
- **채팅방 커버리지 체크** — 주/월간 생성 전, 최신 날짜보다 뒤처진 채팅방이 있으면 확인 대화상자 표시
  - 그대로 유지: 있는 데이터로 요약에 포함
  - 채팅방 삭제: 해당 채팅방의 모든 데이터(messages, daily_summaries, chat_rooms) 삭제
- **새 API** — `GET /rooms/coverage`, `DELETE /rooms/{room_id}`

### 콘텐츠 개선
- **레퍼런스 삭제** — 각 항목 끝의 `> 출처: 채팅방이름` 제거
- **토픽 가중치 + 핵심 섹션 3배 증량** — 개인사업자, 1인개발/바이브코딩, 고충, 아이디어, 리뷰 요청 토픽 우선 분류;
  가장 풍부한 2개 섹션을 3배 분량으로 확장하여 문서 상단 배치

### UI / 워크플로우 개선
- **다중 파일 업로드** — 파일 여러 개 동시 선택 가능, 병렬 미리보기 후 일괄 업로드
- **업로드 모달 내 요약 생성 통합** — 업로드 완료 후 모달 내에서 바로 요약 생성 가능
  파일 없이 "요약 생성 바로 시작" 버튼으로 생성 단계 직행도 지원
- **사이드바 단순화** — 별도 "요약 생성" 버튼 제거, "파일 업로드 / 요약 생성" 버튼 하나로 통합
- **진행 표시 2단계 단순화** — 채팅방별 로그 → "데이터 분석 중... (N/M)" / "다이제스트 생성 중..." / "완료!"
- **주/월간 생성도 SSE 스트리밍** — 배치 진행 상황(일간/주간 자동 생성) 실시간 표시
